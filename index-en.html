Links / Constraints

Each link starts from a sample-local attachment point a_i. Its global point q_i is computed from the current/target sample pose, so the same physical attach point moves with the sample. The other end can be a slider, a fixed anchor, or a joint on an intermediate rigid coupler.

Core Formulas and Algorithm

0. Symbol Table

i: link index, for example A, B, or C.

p: target global position of the sample reference point.

R: target sample rotation matrix.

r: sample reference point in the sample-local frame, usually [0,0,0].

a_i: sample-local attachment point for link i. Its global position changes when the sample moves.

q_i: target global position of sample attachment point i.

b_i: slider base point, or the reference point of the slider rail.

v_i: unit vector of the positive slider direction.

s_i: target absolute slider stroke, not the incremental move.

Δs: required stroke move, Δs = s_target - s_current.

f_i: fixed anchor point in global coordinates.

K_g: intermediate rigid coupler group g.

d_i: local offset of the rod joint on the coupler body.

L_i: link length. For a chain, use L_i1,L_i2,....

residual / e_i: length error, equal to computed distance minus the specified link length.

rod axis: unit vector from the start point of a rod to its sample attachment point. This is not the slider axis.

chain lengths: segment lengths in a multi-link chain, for example 1.4, 1.2, 1.0.

feasible stroke interval: slider stroke range where the multi-link chain can reach the sample point.

middle position: an intermediate sample pose along the motion path. Every middle pose is solved for all actuators.

1. Platform Rigid-Body Transform

The platform target pose is p,R. The attachment point a_i is fixed in the sample-local frame, and the local reference point is r.

q_i = p + R(a_i - r)

q_i is the global target position of the sample-side joint for link i.

Once the sample pose is fixed, every q_i is known. Then each independent link can solve its own actuator degree of freedom.

2. Slider + rod

The slider base is b_i, the positive stroke direction is unit vector v_i, and the rod length is L_i.

m_i(s_i)=b_i+s_i v_i

m_i is the slider-side joint position. s_i is the target absolute slider stroke.

||q_i-m_i(s_i)||^2=L_i^2

3. Analytical Slider Solution

Let c_i=q_i-b_i. Expanding the constraint gives:

s_i^2-2(v_i^T c_i)s_i+(c_i^T c_i-L_i^2)=0

s_i=v_i^T c_i ± sqrt((v_i^T c_i)^2-(c_i^T c_i-L_i^2))

4. Fixed anchor

The fixed anchor is f_i. It has no motor command; it only checks the constraint error:

e_i = ||q_i-f_i|| - L_i

e_i is the residual. If e_i=0, the fixed link is compatible with the target pose.

5. Rod pose

Once the start and end points of a rod are known, the rod pose is also known.

center_i = (start_i + q_i)/2

axis_i = (q_i - start_i)/||q_i - start_i||

start_i is the other end of the link. For a slider link it is m_i; for a fixed link it is f_i.

The GUI reports center, axis, yaw, and pitch.

6. Multi-link chain

If the sample point connects to the slider through a serial chain of links, use segment lengths L_1,L_2,...,L_n.

Maximum reach: Rmax = L_1+L_2+...+L_n

Minimum reach: Rmin = max(0, Lmax - sum(other L))

The distance from slider point m(s) to sample point q must satisfy Rmin <= ||q-m(s)|| <= Rmax.

This gives a feasible stroke interval. Internal chain folding is not unique unless hinge planes or hinge axes are specified.

7. Intermediate coupler

If multiple rods connect to the same intermediate rigid coupler K_g, they share one stroke s_g and must be solved together.

p_i(s_g)=b_g+s_g v_g+d_i

e_i(s_g)=||q_i-p_i(s_g)||-L_i

Stack the residuals from the same coupler group:

F(s_g)=[e_1,e_2,...]^T

J_i=de_i/ds_g. Use a one-dimensional Gauss-Newton update:

(J^T J) Δs = -J^T F

s_g <- s_g + Δs. This is the matrix solve used when several rods are coupled by the same intermediate body.

8. Trajectory and Synchronized Control

Sample several middle poses from current pose to target pose:

p(t)=(1-t)p_0+t p_1

yaw(t), pitch(t), roll(t) are interpolated in the same way.

For every t_k, compute all q_i(t_k), then solve all s_i(t_k).

In actual control this means sending [s_A, s_B, ...] at the same time step. Δprev is the actuator increment from the previous frame.

9. Automatic Lift Height Search

Let h be an extra lift amount: Z_high=max(Z_current,Z_target)+h.

Construct two waypoints:

W1=[X_current,Y_current,Z_high]

W2=[X_target,Y_target,Z_high]

The app starts from h=0, increases by Lift step, and checks Current -> W1 -> W2 -> Target. The first h for which all steps are OK is the minimum feasible lift at the selected search resolution.
