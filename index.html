核心公式与算法

0. 符号表：先看这里

i：第 i 条连杆/link，例如 A、B、C。

p：sample 参考点在全局坐标里的目标位置。

R：sample 的目标姿态旋转矩阵。

r：sample 参考点在 sample 自己局部坐标里的位置，通常是 [0,0,0]。

a_i：第 i 条连杆接在 sample 上的那个点，使用 sample 局部坐标。

q_i：第 i 个 sample 连接点变换到全局坐标后的目标位置，也就是连杆应该接到 sample 的位置。

b_i：slider 的 base 点，也就是滑块轨道的起点/参考点。

v_i：slider 正方向的单位向量。

s_i：slider 的目标绝对位置/stroke，不是移动量。

Δs：slider 需要移动的量，Δs = s_target - s_current。

f_i：fixed anchor 的全局坐标，也就是固定端的位置。

K_g：第 g 个 intermediate coupler / 中间刚体连接块。

d_i：第 i 条杆接在 coupler 上的局部偏移点，使用 coupler 局部坐标。

L_i：第 i 条连杆的长度；如果是多连杆链，则是 L_i1,L_i2,...。

residual / e_i：长度误差，等于“实际算出来的距离 - 设定杆长”。越接近 0 越满足约束。

rod axis：连杆从起点指向 sample 连接点的单位方向向量，不是 slider 轴。

chain lengths：多连杆链中每一段杆的长度，例如 1.4, 1.2, 1.0。

feasible stroke interval：多连杆链能达到 sample 连接点时，slider stroke 可以落入的区间。

middle position：运动路径中的中间 sample 位姿；每个中间位姿都会同时反求所有 actuator。

1. 平台刚体变换

平台目标位姿是 p,R。平台上的局部连接点是 a_i，参考点是 r。

q_i = p + R(a_i - r)

q_i 的意思是：第 i 条连杆在 sample 上的连接点，经过 sample 位姿变换后，在全局坐标里的位置。

一旦 sample 位姿确定，每个 q_i 都确定，所以每条连杆可以独立反求自己的自由度。

2. Slider + rod

slider base 是 b_i，stroke 正方向是单位向量 v_i，连杆长度是 L_i。

m_i(s_i)=b_i+s_i v_i

m_i 是 slider 端点的位置，s_i 是 slider 的目标绝对位置。

||q_i-m_i(s_i)||^2=L_i^2

3. Slider 解析解

令 c_i=q_i-b_i，展开得：

s_i^2-2(v_i^T c_i)s_i+(c_i^T c_i-L_i^2)=0

s_i=v_i^T c_i ± sqrt((v_i^T c_i)^2-(c_i^T c_i-L_i^2))

4. Fixed anchor

固定 anchor 位置是 f_i。它没有 motor command，只检查约束误差：

e_i = ||q_i-f_i|| - L_i

e_i 也叫 residual。若 e_i=0，该固定连杆满足目标位姿；否则该目标位姿与固定约束不一致。

5. Rod pose

每条杆的起点和终点确定后，杆的姿态也确定。

center_i = (start_i + q_i)/2

axis_i = (q_i - start_i)/||q_i - start_i||

start_i 是连杆的另一端：slider link 里是 slider 端点 m_i，fixed link 里是固定点 f_i。

GUI 输出 center、axis、yaw、pitch。

6. Multi-link chain

如果 sample 点不是直接接到 slider，而是经过多根杆串联到 slider，设杆长为 L_1,L_2,...,L_n。

最大可达距离：Rmax = L_1+L_2+...+L_n

最小可达距离：Rmin = max(0, Lmax - sum(other L))

slider 点 m(s) 到 sample 连接点 q 的距离必须满足：Rmin <= ||q-m(s)|| <= Rmax。

这会给出一个可行 stroke 区间；中间每根杆的折叠姿态若没有 hinge plane/axis，则不唯一。

7. Intermediate coupler

如果多根杆接到同一个中间刚体连接块 K_g，它们共享同一个 stroke s_g，不能再完全逐条独立求解。

p_i(s_g)=b_g+s_g v_g+d_i

e_i(s_g)=||q_i-p_i(s_g)||-L_i

把同一个 group 的 residual 叠起来：

F(s_g)=[e_1,e_2,...]^T

J_i=de_i/ds_g，用一维 Gauss-Newton：

(J^T J) Δs = -J^T F

s_g <- s_g + Δs。这就是几根杆被同一个中间块耦合时的 matrix 解法。

8. 轨迹和同步控制

从 current pose 到 target pose 取若干个中间位姿：

p(t)=(1-t)p_0+t p_1

yaw(t), pitch(t), roll(t) 也按同样方式插值。

对每一个 t_k，先算所有 q_i(t_k)，再同时反求所有 s_i(t_k)。

所以实际控制不是“先动 A 再动 B”，而是在同一时间步发送 [s_A, s_B, ...]。GUI 中 Δprev 是每个 actuator 从上一帧到这一帧的增量。

9. 自动找 lift height

设 h 是额外抬高量，Z_high=max(Z_current,Z_target)+h。

构造两个 waypoint：

W1=[X_current,Y_current,Z_high]

W2=[X_target,Y_target,Z_high]

程序从 h=0 开始，每次加 Lift step，检查路径 Current -> W1 -> W2 -> Target。第一个所有 step 都 OK 的 h 就是当前搜索精度下的最小可行抬高量。
